本次测试测得的 OS Integrity (Double Sampling) 延迟与吞吐率均为该方案的 理论最优/最乐观情况 (Best-case Scenario)：

  • 结论：真实环境下的 OS Integrity 延迟将呈 60s 的倍数级增长，吞吐率将进一步折半。而 fustord API 无论文件是否正在写入，其性能表现始终保持恒定。

Test Methodology & Documentation (测试任务详解)

1. 任务定义 (Task Definitions)

  • OS Baseline (Pure find Scan): 执行原生 find 扫描。该方案无法识别正在写入的文件，在 NFS 环境下极易产生“脏读”，数据一致性等级为“低”。
  • OS Integrity (Double Sampling): 模拟 NFS 环境下确保文件完整的真实方案。包含首次采样 -> 强制 {{integrity_interval}}s 物理等待 -> 二次校验。当前测量值为无重试的最优性能。
  • fustord API (Perception Mode): 基于 sensord 实时感知的元数据检索。无需任何物理静默等待即可返回可靠数据。
